# AXIOMS

## AX-MOD-SCRIPTS-001: `local-sync-cron.cjs` must stay alive when `SYNC_SECRET` is absent
The local sync cron driver must NOT exit when `process.env.SYNC_SECRET` is missing. It must log a warning and remain running (currently via `setInterval(() => {}, 1 << 30)`) so that `turbo watch dev:cron` does not treat the process as a crashed task. It must also ignore `SIGHUP` for the same reason. The bearer-token contract with `/api/github/sync-events` is `Authorization: Bearer ${SYNC_SECRET}` over `POST`, with an optional `?fullRefresh=true` query parameter driven by `FULL_REFRESH=true`.

### Applies To
- `scripts/local-sync-cron.cjs`

### Acceptance Checks
- `rg -n "SYNC_SECRET" scripts/local-sync-cron.cjs` shows the env-presence check, the warning log, and the keep-alive `setInterval`.
- `rg -n "SIGHUP" scripts/local-sync-cron.cjs` shows the ignored-SIGHUP handler.
- `rg -n "Authorization.*Bearer.*SYNC_SECRET" scripts/local-sync-cron.cjs` finds the bearer-token header construction.
- manual: run `SYNC_SECRET= pnpm dev:cron` and confirm the process stays alive without spinning.

## AX-MOD-SCRIPTS-002: GitHub-activity backfill writes only to `s3://cenv-public/brian-stoker-github-activity.json`
`populate-github-activity.js` and `populate-github-activity-history.js` must continue to publish to the bucket `cenv-public` at the key `brian-stoker-github-activity.json`, in `us-east-1`, with `ContentType: application/json`, and a payload envelope of `{ data: ActivityDay[], lastUpdated: ISOString }`. Each `ActivityDay` carries `{ date, count, level }` with `level ∈ [0..4]` produced by the `getActivityLevel` mapping (`0` → `0`, `≤3` → `1`, `≤6` → `2`, `≤9` → `3`, else `4`). The bucket is public; sensitive data must never be added to this payload.

### Applies To
- `scripts/populate-github-activity.js`
- `scripts/populate-github-activity-history.js`

### Acceptance Checks
- `rg -n "S3_BUCKET\\s*=\\s*'cenv-public'" scripts/populate-github-activity*.js` matches both files.
- `rg -n "S3_KEY\\s*=\\s*'brian-stoker-github-activity.json'" scripts/populate-github-activity*.js` matches both files.
- `rg -n "getActivityLevel" scripts/populate-github-activity*.js` shows identical bucket thresholds in both files.
- manual: download the produced JSON and verify `level` values are clamped to `0..4`.

## AX-MOD-SCRIPTS-003: Production deploys go through `aws-deploy.sh` and ship logs after `sst deploy`
Production deployment must be invoked via `scripts/aws-deploy.sh deploy` (i.e. `pnpm deploy:prod`). That script must (a) `senvn -f production npx sst deploy --stage production` first, (b) then run `AWS_PROFILE=stoked node scripts/setupLogShipping.cjs`, and (c) treat log-shipping failure as non-fatal (warning printed, exit code 0). The script must accept only `deploy` and `remove` as actions and reject any other argument. AWS account verification is intentionally delegated to `senvn` + `.deploy.json`; do not reintroduce ad-hoc `aws sts` checks in this script.

### Applies To
- `scripts/aws-deploy.sh`
- `scripts/setupLogShipping.cjs`

### Acceptance Checks
- `rg -n "senvn -f production npx sst (deploy|remove) --stage production" scripts/aws-deploy.sh` finds both invocations.
- `rg -n "AWS_PROFILE=stoked node scripts/setupLogShipping.cjs" scripts/aws-deploy.sh` is present and followed by `|| echo`.
- `rg -n "set -euo pipefail" scripts/aws-deploy.sh` is present.
- manual: run `./scripts/aws-deploy.sh foo` and confirm it prints the usage line and exits 1.

## AX-MOD-SCRIPTS-004: `copy-db-schema.ts` is the source of truth for required local Mongo collections
`copy-db-schema.ts` must copy every collection definition and every non-`_id_` index from the `brianstoker-production` database into `brianstoker-local`, and must additionally create the GitHub-activity collections `github_events`, `github_rate_limits`, and `sync_metadata` when missing. The script must hard-fail (`process.exit(1)`) if `MONGODB_URI` is unset. The source/target DB names and the explicit GitHub collections list are observable contract — any change must be reflected in the SC_MODULE doc and validated against the consumers in `pages/api/`.

### Applies To
- `scripts/copy-db-schema.ts`

### Acceptance Checks
- `rg -n "brianstoker-production|brianstoker-local" scripts/copy-db-schema.ts` shows both DB names.
- `rg -n "github_events|github_rate_limits|sync_metadata" scripts/copy-db-schema.ts` shows all three collections.
- `rg -n "MONGODB_URI" scripts/copy-db-schema.ts` shows the env check before `MongoClient` construction.
- `pnpm typescript` passes.

## AX-MOD-SCRIPTS-005: GitHub-secrets configuration must be gated on `gh` workflow scope and `.env.production`
`configure-github-secrets.sh` must (a) `unset GITHUB_TOKEN` so it uses the `gh` CLI's own auth context, (b) refuse to run unless `gh auth status` reports the `workflow` scope, (c) refuse to run if `.env.production` is missing, and (d) translate the `.env.production` variable `GITHUB_TOKEN` into the GitHub repo secret named `GH_TOKEN` (NOT `GITHUB_TOKEN`, which is a reserved GitHub Actions name). The target repository is `brian-stoker/v2.brianstoker.com`.

### Applies To
- `scripts/configure-github-secrets.sh`
- `scripts/show-secrets-values.sh`

### Acceptance Checks
- `rg -n "unset GITHUB_TOKEN" scripts/configure-github-secrets.sh` is present.
- `rg -n "gh auth status .* workflow" scripts/configure-github-secrets.sh` shows the scope check.
- `rg -n "gh secret set GH_TOKEN" scripts/configure-github-secrets.sh` shows the rename.
- `rg -n "brian-stoker/v2.brianstoker.com" scripts/configure-github-secrets.sh scripts/show-secrets-values.sh` is consistent across both files.
- manual: run the script without `.env.production` present and confirm it exits non-zero with the documented error.

## AX-MOD-SCRIPTS-006: Script entry points and their `package.json` recipes are a contract
Renaming, removing, or changing the executable shape of `local-sync-cron.cjs`, `build-prod.js`, `buildIcons.js`, `buildServiceWorker.js`, `reportBrokenLinks.js`, `copy-db-schema.ts`, `list-db-info.ts`, `setupLogShipping.cjs`, or `aws-deploy.sh` is a breaking change that must update the corresponding recipe in the root `package.json` `scripts` block (and any CI workflow that references it) in the same commit. The nested `scripts/package.json` package name `github-activity-scripts` and its `populate-history` recipe are also part of this contract.

### Applies To
- every executable file under `scripts/`
- `package.json` (root)
- `scripts/package.json`

### Acceptance Checks
- `rg -n "scripts/" package.json` lists every script-backed recipe; cross-check that each referenced file exists.
- `pnpm typescript` passes (covers both root and `scripts/tsconfig.json`).
- manual: each renamed script has an updated recipe and any CI workflow that called the old name has been migrated.
